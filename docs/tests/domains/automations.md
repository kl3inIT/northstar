| Behavior | Coverage | Notes |
| --- | --- | --- |
| Migration/JPA/scheduler schema compatibility | Automated | `NorthstarWorkerContextLoadTests` runs Flyway through V27 on PostgreSQL and boots the worker with `ddl-auto: validate` plus db-scheduler beans. |
| Trigger validation and timezone-aware cron compilation | Automated | API invalid-trigger cases plus `AutomationSchedulerCoordinatorTests` verify weekday ordering, six-field cron, and IANA timezone. |
| Definition projection state | Automated | `AutomationSchedulerCoordinatorTests` verifies an unsynced disabled definition is reconciled and its exact schedule version acknowledged. |
| Definition CRUD, optimistic version, types, run-now, and history | Automated | `AutomationControllerIntegrationTests` uses PostgreSQL and MockMvc, including invalid workflow type/trigger and secret-boundary assertions. |
| Deleted definition versus queued manual run | Automated | `AutomationTaskExecutorTests` verifies the worker marks the claimed run `SKIPPED` and never invokes its handler. |
| Morning Brief validation, trust ranking, source-fair selection, source deduplication, deterministic Markdown, partial-source failure, and Staging output | Automated | `MorningBriefHandlerTests` uses fake source/note services, multiple trust tiers, competing candidates from the same source, a failing provider, and tracking-URL variants. |
| Same automation/day output idempotency | Automated | `MorningBriefHandlerTests` verifies a rerun updates the existing named daily note instead of creating another. |
| RSS/Atom secure parsing and official classification | Automated | `RssBriefSourceTests` parses a namespaced Atom entry with external XML access disabled and verifies the official trust tier. |
| Firecrawl budget and cheap-mode request boundary | Automated | `FirecrawlBriefSourceTests` verifies the credit cap limits query count and every request stays at three results, basic proxy, Markdown-only, and no Agent/JSON mode. |
| Generated API trust boundary | Automated | OpenAPI is committed; Hey API emits DTO/SDK/Zod clients and `automation-api.ts` parses list/history responses before UI use. |
| HuggingNews public page-data compatibility | Automated | `HuggingNewsParserTests` decodes captured 2026-07-12 public SvelteKit/devalue feed and story-detail fixtures and pins story identity, rankings, tags, summary, entities, previous story, and source links. |
| Provider-neutral Brief API delegation | Automated | `BriefControllerTests` verifies the REST controller delegates feed/detail reads through `BriefFeedProvider`; generated OpenAPI DTO/SDK/Zod clients validate both responses in the web trust boundary. |
| Runtime type catalog and type-specific creation | Static + runtime | The UI parses `/api/automations/types`, presents registered descriptors before configuration, and routes supported ids through the frontend editor registry. Playwright verified the picker and editor at desktop and 390x844, including initial scroll position and the mobile footer. |
| Assistant/MCP automation tools | Automated | `AutomationToolsTests` pins typed schedule/config normalization, type safety, and invalid-time rejection without runtime mocking. `AutomationControllerIntegrationTests` verifies in-app Assistant discovery. `McpServerIntegrationTests` verifies all tools are published and round-trips Morning Brief creation/listing over streamable HTTP. |
| Settings create/run/history/delete flow | Runtime | Playwright on 2026-07-11 exercised the real API at desktop and 390x844. It caught and verified fixes for nullable contract parsing and history-cache invalidation after deletion. |
| Settings layout and accessibility | Runtime | Desktop/mobile screenshots and accessibility snapshots verify labeled controls, independent dialog scrolling, stable action layout, and no new console errors in the final request flow. |
| Live public-source Morning Brief execution | Deferred verification | Parser/orchestration adapters are covered with fakes. A real scheduled run requires the worker, API, database, and user-owned Firecrawl quota; verification does not spend that quota solely for a build test. |
| `/briefs` two-tab newsroom | Runtime | Playwright on 2026-07-12 exercised the real API and public HuggingNews page data: 68 stories across 3 days, topic filters/TL;DR/day rows, lazy inline detail with key sources, and zero console warnings/errors. It also verified the separate Northstar Brief tab parses the current durable issue into 7 stories and retains Configure Sheet/run controls without mixing either source into the other. |
| HuggingNews inline-detail viewport stability | Static + runtime gap | The row captures its viewport anchor before opening, excludes remote detail content from native scroll anchoring, and restores the row before paint when loading settles. Web typecheck/test/build cover compilation; a browser regression should pin the trigger's `getBoundingClientRect().top` before and after a deliberately delayed detail response. |
| Morning Brief source editor responsiveness | Runtime | Playwright exercised the Sources tab in the full-height Sheet at 1440x1000 and 390x844. Repository/feed/handle fields use full-width rows, the Sheet exactly fits the mobile viewport, and Save/Cancel remain fixed outside the scrolling form. |
| Two live worker processes contending for one due occurrence | Residual gap | Cluster claiming is delegated to db-scheduler and the database primary key; a future deployment smoke test should start two worker containers against one disposable database. |
