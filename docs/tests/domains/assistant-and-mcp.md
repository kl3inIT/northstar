| AI route defaults, typed runtime gateway CRUD, and secret masking | Automated + runtime | `AiPropertiesTests` pins workload-specific model defaults, active-gateway routing, and secret-safe rendering; `AiCredentialCipherTests` pins random AES-GCM envelopes and gateway-bound AAD; `AiRouteSettingsServiceTests` pins default/update/reset precedence. `AiGatewayRuntimeOverrideIntegrationTests` pins one logical OpenAI row, Settings-over-environment credential precedence, and route-preserving reset. | Capability-specific route catalogs | Automated + browser | `AiCapabilityCatalogTests` pins manual-plus-discovered 9Router image targets; `OpenAiCompatibleTextToSpeechGatewayTests` pins optional voice metadata and manual fallback; `SpeechControllerIntegrationTests` pins persisted TTS language fallback. Desktop and 390x844 Playwright checks verify searchable Chat/TTS/STT/Image/Embedding selectors, separate manual catalogs, TTS language filtering, and stable responsive controls. Live 9Router verification returned 14 Chat, 2 working Edge TTS voices, and 3 Image targets while empty STT/Embedding/Web discovery stayed empty rather than leaking another catalog. |
| Runtime-routed embedding, STT, and Realtime | Automated + browser | `RoutedOpenAiModelsTests` verifies selected gateway URL/key/model, 1536 embedding dimensions, and plain-text transcription. `RealtimeSessionControllerTests` pins gateway-selected ephemeral-secret minting and returned WebSocket URL. | Shared gateway web capability routes | Automated + browser | `OpenAiWebSearchProviderTests` pins `/responses` routing; `NineRouterWebProviderTests` pins `/search`, `/web/fetch`, normalized sources/pages, type rejection, and retryable errors. Core tests pin route-aware caching/fallback. Desktop and 390x844 Playwright audits verify one gateway manager plus search/fetch gateway-target controls without duplicate credentials. |
| Web and mobile model selection | Static + browser | Generated OpenAPI client plus web typecheck; Flutter analyze and Assistant service/ViewModel/widget suites; Playwright screenshots at desktop and 390x844 verify route selection, runtime gateway presets, connection feedback, multi-gateway grouping, provider marks, `+`-then-model composer order, and the searchable native Model Selector. |
| Native Assistant message surfaces | Static + browser | Web typecheck/build and browser audit cover AI Elements Attachments, Message Actions, Inline Citation, Sources, Prompt Input, Tool, and Model Selector composition. `UiMessageStreamTests` pins `source-url` and `source-document` frame shapes; `EventEmittingToolManagerTests` pins web/note/file source extraction; the Assistant controller integration test verifies document sources survive history rehydration. Flutter ViewModel/widget tests verify URL and note sources remain typed and visible in the mobile message. |
| On-demand Assistant speech and reusable audio cache | Automated + browser | `SpeechAssetServiceTests` pins validation, persisted cache hits, and no provider call on a hit; `OpenAiCompatibleTextToSpeechGatewayTests` pins direct OpenAI model/voice requests, 9Router discovery, optional voice metadata, and provider failures; `SpeechControllerIntegrationTests` pins synthesis, MP3 serving, request deduplication, and configured-language fallback. | Completed tool workflow remains stable | Runtime | Browser regression on 2026-07-11 ran a real read-only tool turn and verified the workflow ended expanded while the same `[role=log]` node remained connected (`detached=false`, `sameLog=true`); the header still permits explicit user collapse. |
| MCP handshake, tools/list, and tool calls | Automated | `apps/mcp/src/test/java/com/northstar/mcp/McpServerIntegrationTests.java`; includes `create_note` with explicit `status=RESOURCE`. |
